## 修复系统看板网络流量始终为 0
- 问题：网络流量卡一直显示 `0.00 MB/s` 且不变化。
- 根因：
  1. `systeminformation.networkStats()` 首次调用没有 rx/tx 速率基线，返回 `rx_sec`/`tx_sec` 为 `null`；
  2. 原代码只取 `netStats[0]`，在 macOS 等环境可能是 `lo0`（回环）或 `utun*`（隧道），导致无真实流量；
  3. `rx_sec`/`tx_sec` 实际单位是 bytes/sec，但前端直接标成 `MB/s`，小流量也会显示为 `0.00`。
- 修复：
  - `services/systemService.js`：新增 `aggregateNetworkStats`（排除回环 `lo/lo0`，聚合其余所有网卡）和 `computeNetworkSpeed`（模块级 `lastNetSnapshot` 记录上一次总字节数，自行计算 bytes/sec 速率）；接口返回 `{ rxSec, txSec, interfaces }`，单位为 bytes/sec。
  - `src/views/Dashboard.vue`：`netRx`/`netTx` 改为存储 bytes/sec 数字；新增 `formatSpeed` 与 `netRxDisplay`/`netTxDisplay` 计算属性，按 `>=1MB/s` / `>=1KB/s` / 其余 自动切换 `MB/s`/`KB/s`/`B/s`；模板同步替换硬编码 `MB/s`。
- 验证：Node 脚本连调两次 `getSystemResources()`，首次基线为 0，第二次 rxSec≈3500 bytes/sec、txSec≈6400 bytes/sec；`npm run build` 成功；已重启后端。

## 合并「Registry 凭证」到代理管理（消除重复入口）

- 关键技术障碍：go-proxy 管理端口 `getConfig()` 返回的密码是**脱敏的**（********），hubcmdui 搜索服务无法直接复用代理管理里配置的真实密码去申请 token。
- 合并方案（双写，对用户透明）：
  - **删除**：前端 `src/views/RegistryCredentials.vue`、`src/router/index.js` 路由、`src/layout/AdminLayout.vue` 的 nav 项与 `Key` 图标 import、`src/services/index.js` 的凭证 API；后端 `routes/registryCredentials.js` 文件（独立 CRUD 接口随之消失）。
  - **保留内部表** `registry_credentials`（物理表仍在 `database.js`，不再有对外 UI/接口）。
  - 重写 `services/registryCredentialService.js`：去掉对外 CRUD，保留 `getPlainCredential(registryId)`（供搜索 token 流程）与新增 `syncFromGoProxyConfig(cfg)`（按 `REGISTRY_HOSTS` 域名映射把 go-proxy 各 registry 的 `type=basic` 且有用户名的认证加密写入内部表，全量重建）。
  - `routes/goProxy.js` 的 `putConfig` 成功后调用 `syncFromGoProxyConfig(cfg)`，使代理管理保存即同步。
  - 搜索/标签查看的 `fetchWithRegistryAuth` 仍经 `getPlainCredential` 读取，逻辑不变（之前的 GHCR bitnami 等受限仓库能力保留）。
